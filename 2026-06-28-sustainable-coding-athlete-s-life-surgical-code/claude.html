Why this holds

02 / RATIONALE
01

Human Capacity is the Bottleneck

All advisors agree AI's infinite potential is not matched by human cognitive limits; judgment, review, and recovery are the true constraints.

02

The Athlete Frame is Foundation

The "Athlete Frame" is the essential gatekeeper, ensuring sleep, movement, and presence enable effective coding — not just maximize output.

Surgical Scope Prevents Debt

Bounding work into narrow, verifiable "surgical operations" is critical for managing cognitive load and preventing unmanageable technical debt.

The Protocol

04 / ACTION PLAN
Gate01

Start every day with a "Human Capacity Scope Gate" — assess recovery and context to set daily coding limits.

Bound02

Define each "Surgical Operation" with a hard diff budget (<150 lines target) and explicit exit criteria before starting.

Block03

Implement dedicated deep-work blocks (90–150 min) with hard stops — no coding after 9:30 PM — and mandatory movement.

Track04

Track "Human Load" metrics — clean days, late-coding violations, unreviewed diffs, movement — alongside code output.

Sustain05

Proactively address technical debt and schedule dedicated time for continuous learning and skill development.

Tensions & Tests

05 / SCRUTINY

Caveats to watch

  • The "Athlete Frame" can become an anxious ritual if over-indexed on sensor data.
  • Individual "Athlete Frame" needs must be reconciled with team dynamics and organizational culture.
  • Effective automated verification is key to safely increasing throughput in "coach-mode" orchestration.

What would change our mind

If "heroic all-day AI coding" consistently produced demonstrably superior long-term product quality, faster sustainable innovation, higher developer retention, and lower operational costs than the Athlete-Engineer approach — rendering human limitations and technical debt negligible.
